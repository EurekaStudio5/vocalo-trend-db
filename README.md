# ボカロトレンドDB

YouTube上のボカロ曲トレンドを毎日自動収集して可視化するWebサイト。

## 使い方

| やること | 方法 |
|---|---|
| サイトを見る | `サイトを起動.bat` をダブルクリック（ブラウザが自動で開きます） |
| データを手動更新 | `データ更新.bat` をダブルクリック（数分かかります） |
| 自動更新 | Windowsタスクスケジューラに登録済み（毎日 06:00、タスク名: `VocaloTrendDB_DailyUpdate`） |

## 機能

- **週間/月間/年間/全期間 楽曲ランキング** — 期間内に投稿されたボカロオリジナル曲を再生数順に表示（各100位）。サムネクリックでサイト内再生（YouTube埋め込み）。ボーカル別フィルタ・スコア順/伸び率順ソート・検索対応。
- **VocaScore（曲単体評価スコア）** — 登録者数・投稿日時の影響を補正した0〜100の独自評価。好感度（高評価率, 40点）＋拡散力（再生数/登録者数, 30点）＋勢い（時間正規化再生数, 30点）。取得できない指標は除外して100点満点に換算。各曲のバッジにホバーで内訳表示。
- **トレンド分析（年間）** — 使用ボーカル別の曲数/平均再生数、伸びやすい動画の長さ、タイトル頻出ワード、新星ボカロP（登録者比で伸びている曲）。
- **ボカロPランキング（100軸）** — 「登録者数キング」「テト愛」「ジャスト3分職人」「失踪疑惑」など、分析系〜ネタ系まで100種類の軸でPをランキング。カテゴリ絞り込み＋🎲ランダム表示。各Pの直近動画30本・チャンネル概要・ランクイン実績から自動計算。
- **ボカロP名鑑** — 登録者1万人以上のボカロPデータベース。毎日の更新で自動的に新しいPが追加・蓄積されます。

## 仕組み

- `update_data.py` がYouTubeの公開検索結果ページからデータを取得（APIキー不要）
- ランキング上位曲は動画ページから高評価数・正確な再生数・投稿日時も取得（`data/stats_cache.json` にキャッシュ、週間=毎日/月間=3日/年間=7日/全期間=14日で再取得）
- 取得データは `data/videos.json`（weekly/monthly/yearly/alltime）等に保存、`data/history/` に日次スナップショットを蓄積
- 各ボカロPの全ボカロ楽曲リストは `data/channels/{チャンネルID}.json` に保存（詳細パネルで使用）
- フロントエンドは静的HTML+JSで `data/*.json` を読むだけ
- `update_task.bat` がタスクスケジューラから毎日実行される

## 毎日の自動更新で行われること（ローテーション方式）

毎日の処理量を一定（約10分）に保つため、重い処理は数日かけて全Pを一巡するローテーションで実行:

| 処理 | 頻度 |
|---|---|
| 楽曲ランキング収集（週/月/年/全期間） | 毎日 |
| ランクインしたPの名鑑への自動追加 | 毎日 |
| 名鑑Pの新曲チェック（直近動画） | P毎に3日ごと |
| 名鑑Pの基本情報更新（登録者数等） | P毎に4日ごと |
| 名鑑Pの概要情報（開設日・累計再生） | P毎に7日ごと |
| 名鑑Pの全曲リスト再取得 | P毎に14日ごと（新曲は毎回の新曲チェックで即マージ） |
| シードリストからのP検索 | シード毎に7日ごと（25件/日） |
| チャンネル発見クエリ（「ボカロP」等で一括発見） | 3クエリ/日でローテーション |

新しく発見されたチャンネルは、直近動画のボカロ曲数を検証してから名鑑に載る（歌い手・ゲーム実況等は自動排除）。

## Web公開について

サイトは完全な静的構成（HTML+JS+JSON）なので、`data/` ごとアップロードすればそのまま公開できます。
推奨: GitHub Pages + GitHub Actions（cronで毎日 `update_data.py` を実行して `data/` をコミット）、
または Netlify / Cloudflare Pages。サーバーサイド処理は不要です。

## 注意

- YouTubeのページ構造変更でデータ取得が止まる可能性があります。その場合は `update_log.txt` を確認してください。
- 再生数・登録者数はYouTube表示上の概数（「12万回」等）です。
- 非公式の個人用ファンサイトです。
